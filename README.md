# Редактируемая таблица данных

Тестовое приложение для работы с редактируемой таблицей данных с поддержкой различных типов ячеек и сохранения изменений.

## Демо

https://open-my-game-test.vercel.app/

## Технологии

- **React 19**
- **TypeScript**
- **Vite**
- **Ant Design**
- **TanStack Query (React Query)**
- **SCSS**
- **Day.js**
- **Axios**

## Основной функционал

- **Кастомная таблица** - полностью реализованная с нуля таблица с модульной архитектурой:
  - Компонентная структура: `TableHeader`, `TableBody`, `TableRow`, `TableCell`
  - Система хуков для управления состоянием: `useTableEditing`, `useTableData`, `useLazyTableData`, `useSaveBatchMutation`, `useInfiniteScroll`
  - Контекстная система конфигурации через `TableConfigProvider` для передачи колонок и обработчиков сохранения
  - Кастомные редакторы ячеек для каждого типа данных с адаптерами
  - Мемоизация компонентов для оптимизации производительности
  - Система отслеживания оригинальных значений для определения изменений (dirty state)
  - Обработка ошибок валидации с визуальной индикацией
  - Поддержка клавиатурной навигации (Enter для сохранения, Escape для отмены)
- **Редактирование ячеек** - клик по ячейке для редактирования
- **Различные типы данных**:
  - string
  - number с поддержкой целых и дробных значений
  - enum с выпадающим списком
  - Даты и время (timestamp)
  - Массивы - теги
  - Объекты - JSON редактор
- **Ленивая загрузка данных** - автоматическая подгрузка при прокрутке (infinite scroll)
- **Сохранение таблицы** - сохранение всех изменений одной кнопкой (отправка на бек)
- **Валидация данных** - проверка корректности введенных значений
- **Отслеживание изменений** - визуальная индикация измененных ячеек
- **Адаптивный дизайн** - минимально для разных девайсов

## Как запустить приложение

### Требования

- Node.js (версия 18 или выше)
- npm или yarn

### Установка зависимостей

```bash
npm install
```

### Запуск в режиме разработки

```bash
npm run dev
```

## Примечания
При наличии большего времени можно было бы:
- Добавить дополнительную валидацию исходя из требований
- Добавить виртуализацию для больших объемов данных (есть лейзи и кеширование)
- Добавить переводы (i18n)
- Улучшить дизайн: закрепить хедер для удобства, улучшить отображение JSON, добавить больше стилизации и т.д.
- Добавить фильрацию, сортировку
- Добавить бек
- При наличии багов, пофиксить их
